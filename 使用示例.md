# RSA 使用示例

## 示例1：基础加密解密

### 场景
小明想给自己的日记加密

### 步骤
1. 运行程序，生成密钥对（级别1）
2. 加密文本："今天天气很好"
3. 得到密文：20170 20170 20170 22825 22825 22825 ...
4. 保存密文
5. 解密时输入密文，还原原文

## 示例2：给朋友发送加密消息

### 场景
小明想给小红发送加密的生日祝福

### 小红的操作（接收方）
1. 运行程序，生成自己的密钥对
2. 将 `public_key.txt` 发送给小明
3. 保管好 `private_key.txt`

### 小明的操作（发送方）
1. 收到小红的 `public_key.txt`
2. 将文件内容手动创建为 `public_key.txt`
3. 运行程序，选择"5. 重新加载密钥"（只会加载公钥）
4. 选择"2. 加密文本"
5. 输入："生日快乐！"
6. 复制密文发送给小红

### 小红解密（接收方）
1. 运行程序（会自动加载自己的私钥）
2. 选择"3. 解密文本"
3. 粘贴小明发来的密文
4. 看到解密后的祝福："生日快乐！"

## 示例3：加密重要账号信息

### 场景
保存网站密码到文本文件中

### 操作
1. 生成密钥对（级别3，高安全）
2. 备份 `private_key.txt` 到U盘
3. 加密账号密码信息
```
网站：example.com
用户名：user123
密码：MySecret@2024
```
4. 保存密文到文件 `accounts_encrypted.txt`
5. 需要时解密查看

## 安全提示

### ? 正确做法
- 定期备份密钥文件
- 私钥单独保存在安全位置
- 重要信息选择高安全级别
- 加密前检查密钥是否正确

### ? 错误做法
- 不要通过网络发送私钥
- 不要使用低安全级别加密重要信息
- 不要忘记备份密钥
- 不要在公共电脑上使用

## 密文格式说明

### 正确的密文格式
```
123 456 789 101 102 103
```
- 数字之间用空格分隔
- 每个数字代表一个加密后的字符

### 错误的密文格式
```
123456789  （没有空格）
123,456,789  （用逗号分隔）
123-456-789  （用减号分隔）
```

## 实际应用建议

### 适合RSA加密的场景
- 密码管理
- 私人笔记
- 通信加密
- 敏感文档

### 不适合RSA加密的场景
- 大文件（推荐使用对称加密）
- 实时通信（延迟较高）
- 频繁加解密（性能考虑）

## 进阶技巧

### 技巧1：使用不同密钥对分类管理
- 工作相关：使用密钥对A（级别2）
- 个人隐私：使用密钥对B（级别3）
- 临时文件：使用密钥对C（级别1）

### 技巧2：密钥轮换
- 每3-6个月更换一次密钥
- 旧密钥保留用于解密历史数据
- 新数据使用新密钥加密

### 技巧3：密钥备份策略
- 主密钥：保存在电脑
- 备份1：保存在U盘
- 备份2：保存在云盘（加密保护）
- 备份3：打印纸质版存放保险柜
